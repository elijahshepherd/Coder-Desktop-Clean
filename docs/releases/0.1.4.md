# Fix missing diff module in packaged app

Coder Desktop 0.1.4 fixes a packaging bug that caused `Cannot find module 'diff'` crashes on fresh installs.

---

## Highlights

- Fixed `Cannot find module 'diff'` error on fresh Windows/macOS installs
- Root cause: `electron-builder.yml` incorrectly excluded ALL `node_modules` from the ASAR archive, stripping production dependencies like `diff`
- Verified production build completes successfully and `diff` is now included in the packaged app

## Fixed

- Removed erroneous `!**/node_modules/**` exclude from `electron-builder.yml` — electron-builder's default dependency collection now properly includes only `dependencies` (`diff`, `lucide-react`, `react`, `react-dom`) while auto-pruning `devDependencies`
- CI workflow now triggers on `master` branch (default branch) in addition to `main`, so packaging regressions are caught earlier

## Other Changes

- Confirmed Windows NSIS installer and macOS ZIPs build and run correctly with the fix

---

## Install

**Windows**: Download `Coder-Desktop-0.1.4-setup-win-x64.exe` and run it. The installer supports automatic updates.

**macOS (Intel)**: Download `Coder-Desktop-0.1.4-mac-x64.zip`, extract, and move `Coder Desktop.app` to Applications.

**macOS (Apple Silicon)**: Download `Coder-Desktop-0.1.4-mac-arm64.zip`, extract, and move `Coder Desktop.app` to Applications.

All artifacts include SHA256 checksums in the release manifest.

- This is a patch release fixing a build/packaging issue only
