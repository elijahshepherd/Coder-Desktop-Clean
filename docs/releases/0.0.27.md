# Release Summary

## Overview

Coder Desktop 0.0.27 focuses on correcting install version reporting and making Windows update installation more reliable. This update matters because users must be able to install a new release and immediately see the same version inside the running desktop app.

This version also improves cleanup around update downloads. Coder Desktop now removes older Coder Desktop release artifacts and app-owned staging folders after startup so the user's Downloads folder does not keep filling with previous installers, portable builds, ZIPs, and update files.

---

# Major Highlights

> Include at least 3 highlights whenever possible.

## Correct Installed Version Display

### The renderer version now comes from the package version instead of an old hard-coded file.

* Replaced the stale shared version value that still showed `0.0.19`.

* Added a version sync script that writes the renderer version from `package.json`.

* Runs version sync before development, tests, typecheck, packaging, and release builds.

* Keeps the Settings version pill aligned with the installed app package.

* Reduces confusion after installing a newer release.

---

## Verified Windows Relaunch

### The Windows updater now relaunches only the executable that reports the expected installed version.

* Checks candidate executable file versions before relaunching.

* Skips old executable paths that still exist on the machine.

* Logs mismatched executable versions to the local update log.

* Removes staged installer artifacts after a successful verified relaunch.

* Falls back to opening the installer visibly if silent installation does not produce the expected version.

---

## Safe Download Cleanup

### Coder Desktop now cleans old app downloads without touching unrelated files.

* Removes older Coder Desktop setup installers from the user's Downloads folder.

* Removes older Coder Desktop portable executables and ZIP backup packages.

* Removes older Coder Desktop blockmap, DMG, MSI, and AppImage artifacts.

* Removes app-owned update staging folders from Downloads and the temp directory.

* Keeps current-version Coder Desktop artifacts and unrelated downloads in place.

---

# Technical Improvements

> Include at least 2 technical improvements unless the release only contains one.

## Generated Shared Version Metadata

The shared renderer version file is now generated from `package.json` by `scripts/write-shared-version.mjs`.

This improves release reliability by making the package version the source of truth for both the packaged app and the user-facing version display.

---

## Version-Matched Relaunch Helper

The Windows installer helper now resolves a matching executable path before it starts Coder Desktop again.

This improves update reliability by preventing stale install paths from reopening an older build after a newer installer succeeds.

Fixed update staging cleanup so successful Windows updates remove temporary installer artifacts after relaunch.

Fixed documentation and release metadata that still identified `0.0.26` as the current version.

# Upgrade Guidance

## Before Updating

* Close any running Coder Desktop windows before installing the new version.

* Use the official GitHub Release asset for `0.0.27`.

* Keep a backup of important work outside the app before testing a new desktop build.

## After Updating

* Open Coder Desktop and confirm the app reports `Version 0.0.27`.

* Check that old Coder Desktop installers or ZIPs from earlier versions are removed from Downloads.

* Use the update checker to confirm the same version is not offered again.

# Closing Notes

Coder Desktop 0.0.27 improves one of the most important release expectations: the version a user installs should be the version the app shows when it opens. The release also makes the Windows updater more careful about which executable it starts after installation.

This update improves platform maturity by reducing stale install confusion and keeping old Coder Desktop downloads from piling up over time. The cleanup behavior is intentionally safe and narrow so it helps users without touching unrelated files.

Future releases should continue strengthening installation, update verification, and release packaging so Coder Desktop feels like a dependable desktop product rather than a collection of manual download steps.
