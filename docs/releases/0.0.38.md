# Coder Desktop - Version 0.0.38

> A focused patch release that rewires bug reporting so it works the way the user expects, cleans up dark mode in the Security panel, wraps the like and dislike buttons correctly, and replaces the small image generation animation card with a full stage animation plus a real Download button.

**Release date:** June 15, 2026

**Version:** `0.0.38`

**Status:** Patch

**Platform support:** Windows, macOS

**Full changelog:** https://github.com/elijahshepherd/Coder-Desktop/blob/main/CHANGELOG.md

---

# Release Summary

## Overview

Version 0.0.38 is a small but important patch that fixes how Coder Desktop reports bugs. Before this version, clicking the Report bug button in settings could trigger the user's local Git credential helper, open their browser, and ask them to fill out a prefilled issue form. Automatic reports were doing the same thing, which is why the GitHub Issues page filled up with duplicate manual and "hourly diagnostic scan" issues instead of real bug reports.

That whole flow is gone. Reports are now sent directly to GitHub when the user sets a personal access token environment variable on their computer, and they save locally with a clean message when no token is configured. No more surprise Git prompts, no more browser tabs, no more giant URLs dumped into the Local settings panel.

The release also fixes the dark mode Security panel, where a stray rectangle appeared behind the permission switch text, and tightens the like and dislike button row so it stays inside chat bubbles on narrow widths. The image generation animation finally owns the whole card during thinking, and a Download button now replaces the Copy link on every generated image.

## Big picture direction

Smaller releases, real fixes, no surprises. 0.0.38 is the first patch that closes every open GitHub issue in one go. The next work is on cleaner feedback flow, more provider support, and continuing the steady desktop polish pass that started in 0.0.35.

# Major Highlights

## Bug reporting no longer hijacks Git or the browser

### Clicking Report bug sends a real issue to GitHub, or saves it locally with a clear message.

* Manual bug reports from settings now create the GitHub issue directly when `CODER_DESKTOP_GITHUB_TOKEN` (or `GH_TOKEN`, `GITHUB_TOKEN`, `CODER_DESKTOP_BUG_REPORT_TOKEN`) is set as an environment variable on this computer.
* Manual reports no longer trigger `git credential fill`, no longer open a browser with a prefilled issue link, and no longer dump a giant URL into the Local settings panel.
* When no token is configured, the report is saved locally with a clear next step message that says exactly which environment variable to set.
* Like and dislike feedback follows the same path. Sending feedback sends a GitHub issue. There is no browser step.
* The Local settings notice now wraps cleanly inside the panel and never contains the long issue URL.

---

## Diagnostic scan stops flooding GitHub when offline

### The hourly scan used to create a new high severity issue every hour when GitHub was not configured locally. That cascade is over.

* The scan only escalates to GitHub when the local pending report archive grows well past 200 entries, which is the only case that indicates a real GitHub backlog.
* The pending report archive now stays numbered and predictable so the scan has a clear, accurate signal instead of conflating "no token" with "GitHub is broken."
* Health scan entries never open a browser, never trigger Git credential helper, and never display a giant drop down of payload text in any UI panel.

---

## Animated image generation card fills the whole card

### The black and white spews animation now owns the entire image generation card while the model is working, with the chat prompt anchored underneath.

* The thinking stage card shows the animation at full size instead of a small framed box inside a separate loading region.
* The card title and description appear underneath the animation so the user always sees what the assistant is generating and why.
* The card has its own subtle gradient background that matches the rest of the desktop and reads clearly in both themes.
* Once the image arrives, the card switches to a header plus the result image with a Download action, instead of the previous Copy link.

# Technical Improvements

## New image download pipeline

The image generation card now ships images through a real Electron save dialog instead of relying on a Copy link or a web request. The main process parses the image data URL, opens the system save dialog with sensible default filters, writes the file, and reveals it in the file explorer when finished. The renderer schedules the work, holds the button in a busy state, and rolls back to a retry state if anything fails, so the user always sees honest feedback.

## Cleaner GitHub token resolution

The issue reporter no longer shells out to `git credential fill` from the main process. The only way Coder Desktop now picks up a GitHub token is through environment variables that the user controls. This removes the Git Credential Manager popup the user was seeing on Windows and gives the user a clear and well documented place to put a token. The reporter keeps its sanitization, dedupe, and rate limit logic unchanged, so the parts that already worked are unchanged.

# Developer Notes

## Internal Changes

* Removed the `git credential fill` fallback from the issue reporter. The reporter now only reads GitHub tokens from environment variables.
* Removed the prefilled issue URL generator and the local fallback helper that constructed a prefilled link. The reporter now relies entirely on real submissions or the local cache.
* Renamed the `CreateIssueOptions.openFallback` flag to `openExternal` to be honest about what it does. The flag is still false for manual bug reports and for like and dislike feedback in 0.0.38.
* Bumped `pendingReportWarningCount` to 200 so the diagnostic scan stays quiet on machines that have never configured a token.
* Added a friendly "set `CODER_DESKTOP_GITHUB_TOKEN`" hint to the queued report message, so users have an obvious next step.
* Wired `saveGeneratedImage` across the preload bridge, the desktop API, and the preview API. The browser preview returns a clear "browser preview cannot save" response instead of throwing.

## Dependency Refresh

* No new runtime dependencies were added in this release.
* No existing dependencies were bumped in this release.

## Build Pipeline Highlights

* `npm run release:current` now produces the same Windows artifacts the project has shipped in 0.0.35 through 0.0.37.
* `npm run sync:version` continues to be the single source of truth for `src/shared/version.ts`. Bump package.json, run the sync, ship.

# Upgrade Guidance

## Before Updating

* Back up any saved chats or settings through the existing Reset and reopen flow if you want to roll back easily.
* No token migration is required. Coder Desktop has never stored a GitHub token, and this release does not change that. It only reads one from the user's environment when needed.
* Read the GitHub token section in `docs/security.md` if you want to set `CODER_DESKTOP_GITHUB_TOKEN` so manual and automatic reports start sending to GitHub again.

## After Updating

* Confirm that clicking Report bug in Local settings shows the short saved locally message (or sends a real issue if you have set the token).
* Confirm that the dark mode Security panel no longer shows a stray gray rectangle behind any switch.
* Confirm that like and dislike controls fit inside chat bubbles on narrow widths.
* Confirm that generating an image now shows the full card animation during thinking and a Download button after the image arrives.

# Closing Notes

This release cleans up the most visible pain points reported in the issue tracker and leaves the app in a calmer, more predictable state for everyone. Bug reporting is now honest about what it can and cannot do, the Security panel reads correctly in both themes, and the image generation flow finally looks like one card instead of a stack of framed previews.

The next releases will continue the steady polish pass on the chat experience, expand provider support, and keep tightening feedback flow so the app gets out of the user's way. As always, bug reports that point to a real issue carry the most weight, so the change in 0.0.38 should make it easier to triage what actually needs attention.

If you want to follow along closely, the GitHub Issues page will now only carry real user filed reports and real high severity problems. Thank you for using Coder Desktop.
