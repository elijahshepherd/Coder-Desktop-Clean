# Coder Desktop - Version 0.0.39

> A feature release that redesigns the image generation card with collapsible results, edit and download actions, client-side 4x upscale, full-stage thinking animation, updated logos, system prompt improvements for truthful project identity, AI continuation after image generation, notification deduplication, and accurate provider error reporting.

**Version:** `0.0.39`

**Status:** Feature

**Platform support:** Windows, macOS

**Full changelog:** https://github.com/elijahshepherd/Coder-Desktop/blob/main/CHANGELOG.md

# Release Summary

## Overview

Version 0.0.39 focuses on improving the image generation experience, cleaning up notification behavior, and making the AI assistant smarter about continuing after tool actions. The image generation card has been fully redesigned: it now collapses by default to keep the chat feed compact, includes an Edit button for iterating on generated images, a Download button with real state feedback, and a client-side 4x upscale toggle. The thinking stage now fills the entire card with the animation, and the error stage has a clean full-width layout.

The system prompt now truthfully identifies Coder Desktop as an open-source project by Elijah Shepherd running locally on Windows with Electron and React. The "stop after image generation" bug is fixed: the assistant now continues talking after generating images, describing the result or taking the next user-requested action instead of going silent.

## Big picture direction

Continued polish on the image generation pipeline, notification behavior, and system prompt accuracy. The steady desktop refinement that started in 0.0.35 continues with this release, adding real utility to generated images (edit, download, upscale) and fixing the assistant loop so it no longer stops mid-conversation after generating visual output.

# Major Highlights

## Redesigned image generation card

### The image card now collapses by default, includes Edit and Download buttons with real state feedback, and shows the thinking animation at full stage.

- Cards collapse by default when `autoCollapseImageCards` is enabled, keeping the message feed clean after multiple images are generated.
- A ChevronDown toggle button with "Show"/"Hide" label lets the user expand or collapse each image card.
- Each generated image now has an Edit button (Pencil icon) wired through an `onEditImage` callback that passes the source image, prompt, provider, and model for iteration.
- The Download button shows real states: "Saving", "Saved", "Retry download", with the correct disabled state during the save operation.
- The thinking stage fills the entire card with the black-and-white-spews animation and the status caption anchored at the bottom, replacing the old two-row split layout.
- The error stage has its own clean full-width layout with the error icon, title, description, and error message.

### Client-side 4x image upscale

- When enabled, generated images are upscaled 4x using canvas `createImageBitmap` with `imageSmoothingQuality: "high"`.
- This is an honest client-side bilinear upscale, not a server-side AI upscaler.

---

## AI continues after image generation

### The assistant loop no longer breaks after generating images. The AI now describes the result or continues with the next action.

- Both the `chat:send` handler and the `continueAssistantFromLatestState` function were fixed to continue the tool loop after image generation plans complete.
- The `break` that previously ended the loop after any image plans is removed, so the assistant gets a chance to respond to image results.
- The system prompt now tells the assistant to "continue with a brief description of the generated image" after the image card completes.

---

## Accurate provider error reporting

### The `triedModels` field in provider error blocks now only lists models that were actually attempted during the fallback chain.

Previously, every model in the `candidateModels` array (including all configured fallbacks) was listed as "tried" even when the fallback loop broke early due to a non-retryable error. The fix tracks only models that were actually reached in the loop, giving users accurate error messages.

---

## Notification deduplication

### Completion notifications now fire exactly once per completed message.

Two separate `useEffect` hooks were watching overlapping state, causing duplicate native notifications for completed assistant messages. The fix merges the hooks into one with a `lastNotifiedRef` guard that prevents duplicate notifications for the same message ID.

---

## System prompt improvements

### Coder Desktop now truthfully identifies itself as an open-source project, explains its local architecture, and clarifies it is not affiliated with Coder remote workspace.

- Identity section updated with project facts: open-source repository at `github.com/elijahshepherd/Coder-Desktop`, built with Electron and React, runs on Windows, uses user's own API keys.
- Image generation tool section changed from "stop" to "continue with a brief description" after image card completes.
- Internet tools and brand voice sections both include a "search again on fresh question" rule to prevent stale search results.
- Added "verify factual claims before stating them confidently" rule.

# Technical Improvements

## Clear local data on version update

A new `clearLocalDataOnVersionUpdate` personalization setting detects when the app version has changed between launches and resets local state when enabled. The setting is off by default (`false`). The `AppState` interface now includes a `lastVersion` field that stores the version at last launch, and the `DesktopStore.create` method compares this against the current `appVersion` from `shared/version.ts`.

# Developer Notes

## Internal Changes

- Removed the `if (requestedImagePlans.length > 0) { break; }` early exit from both the `chat:send` loop and `continueAssistantFromLatestState` so the assistant continues after image generation.
- Added `attemptedModels: string[]` tracking in `createAssistantResponse` to accurately report which models were tried.
- Merged two completion-detection `useEffect` hooks in `ChatWorkspace.tsx` into one with a `lastNotifiedRef` guard.
- Added `lastVersion` to `AppState` and `createDefaultState` for version change detection.
- Added version comparison logic in `DesktopStore.create()` that clears local state when `clearLocalDataOnVersionUpdate` is enabled and the version differs.
- Added `import { appVersion } from "../shared/version"` to `store.ts`.

## Dependency Refresh

- No new runtime dependencies were added in this release.
- No existing dependencies were bumped in this release.

## Build Pipeline Highlights

- `npm run release:current` continues to be the single release command.
- The `sync:version` script (run automatically before `typecheck`, `test`, `build`, `dist`, and `release:current`) keeps `src/shared/version.ts` in sync with `package.json`.

# Upgrade Guidance

## Before Updating

- Back up any saved chats or settings through the existing Reset and reopen flow if you want to roll back easily.
- No data migration is required. State is preserved across version updates unless `clearLocalDataOnVersionUpdate` is enabled.

## After Updating

- Confirm that image generation cards collapse by default and can be expanded via the "Show" button.
- Confirm that each generated image has both an Edit and a Download button.
- Confirm that the thinking animation fills the entire card.

- Confirm that provider errors only list models that were actually attempted.
- Confirm that native completion notifications fire only once per message.
