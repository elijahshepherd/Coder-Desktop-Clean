# Coder Desktop - Version 0.0.40

> A feature release that adds the Sub Apps modal with Image Studio coming-soon card, auto-accept image generation toggle, license rewrite (read-only source, fork/publish with credit), preference-card accent borders, bug report text fix, and maintenance system scaffolding.

**Release date:** June 16, 2026

**Version:** `0.0.40`

**Status:** Feature

**Platform support:** Windows, macOS

**Full changelog:** https://github.com/elijahshepherd/Coder-Desktop/blob/main/CHANGELOG.md

---

# Release Summary

## Overview

Version 0.0.40 introduces the Sub Apps modal as a launchpad for future mini-applications within Coder Desktop, starting with an Image Studio card. The license has been rewritten to clarify the read-only source status and allow forking with credit. The Settings panel gets an auto-accept toggle for image generation with a warning about model and cost responsibility. The bug report queued text no longer asks users to set an environment variable, and the feedback action label no longer gets stuck showing "Sending feedback."

## Big picture direction

This release establishes the Sub Apps framework as a dedicated space for features that go beyond the core chat interface. The Image Studio card signals where image editing and iteration tools will live in a future release. The auto-accept image generation toggle gives users explicit control over when the AI can trigger image generation autonomously.

# Major Highlights

## Sub Apps modal with Image Studio

### A new modal provides a dedicated launchpad for sub-applications within Coder Desktop, beginning with Image Studio.

- The Sub Apps button sits in the sidebar footer above Settings, using a Grid3X3 icon.
- Clicking it opens a modal with a grid of application cards.
- The first card is Image Studio, marked as "coming soon," indicating where future image editing and iteration features will live.

### CSS

- `.sub-apps-modal`, `.sub-apps-grid`, `.sub-app-card`, and `.sub-app-icon` styles added.
- The sidebar gains a footer section to host Sub Apps and Settings buttons.

---

## Auto-accept image generation toggle

### A new personalization setting controls whether the AI can trigger image generation without explicit approval.

- The toggle is in the Image Generation section of the Settings panel.
- When disabled (default), the AI will ask for permission before generating images.
- When enabled, a warning is shown: "When enabled, AI can trigger image generation without asking for permission each time. You are responsible for the model and cost usage."
- The `PermissionButton` component now supports an optional `warning` prop rendered as a `.preference-warning` paragraph.

---

## Preference-card accent borders

### The preference cards in the Settings panel now use `var(--accent-border)` and `var(--accent)` for their border colors, responding to the current tone accent.

- `.preference-card` border changed from hardcoded graphite to `var(--accent-border)`.
- `.preference-card:hover` border changed to `var(--accent)`.
- `.settings-root .permission-switch.compact.active` border and background now use `var(--accent)` and `var(--accent-soft)` instead of the previous colorless mix.
- Update notice text elements use `.update-notice-error-message` with accent border and muted text.

---

## Bug report text fix

### The queued bug report message no longer asks users to set an environment variable.

- `formatQueuedMessage` changed back from "Set CODER_DESKTOP_GITHUB_TOKEN..." to "Will retry the next time the app talks to GitHub."
- This matches the pre-v0.0.25 behavior and avoids confusing users.

---

## Feedback action label fix

### The "Sending feedback" label no longer gets stuck when feedback is queued.

- The feedback actions component now shows "Saved locally" for reports with a queued status and no `issueUrl`, instead of showing the stuck "Sending feedback" indeterminate state.

---

## License rewrite

### The LICENSE was rewritten to clarify the read-only source status and allow forking with credit.

- Key terms: source is read-only, you can fork, clone, and publish with credit to Elijah Shepherd.
- Commercial use at 1K+ consecutive daily users requires 50% revenue share with Elijah Shepherd.
- This replaces the previous license that was never formally updated for the public release.

# Technical Improvements

## Maintenance system scaffolding

### The AppState interface now includes a `MaintenanceState` field, and a `maintenance.ts` module provides the parser and lock-file management for the upcoming AS (Automatic Stoppers) system.

- `MaintenanceState` type added to `shared/types.ts`.
- `maintenance.ts` module with `parseMaintenanceBody`, `isMaintenanceIssue`, `readActiveMaintenance`, `setMaintenanceLock`, `clearMaintenanceLock`.
- The maintenance lock file system reads from `{dataPath}/maintenance/active.lock` and supports ESID/OF/AL/NOTE fields.

# Developer Notes

## Internal Changes
- Added `SubAppsModal.tsx` with grid layout, coming-soon cards, and portal-based modal render.
- Added `sub-apps:open` custom event dispatch from sidebar button, handled in `App.tsx`.
- Added `autoAcceptImageGeneration` to `PersonalizationSettings` and `sanitizePersonalizationSettings` in store.

- Added `attemptedModels` tracking to `createAssistantResponse`.

## Dependency Refresh
- No new runtime dependencies were added in this release.
- No existing dependencies were bumped in this release.

## Build Pipeline Highlights
- `npm run release:current` continues to be the single release command.
- `sync:version` (run before typecheck, test, build, dist, release:current) keeps `src/shared/version.ts` in sync with `package.json`.

# Upgrade Guidance

## Before Updating
- Back up any saved chats or settings through the existing Reset and reopen flow if you want to roll back easily.
- No data migration is required. State is preserved across version updates unless `clearLocalDataOnVersionUpdate` is enabled.

## After Updating
- Confirm the Sub Apps button appears in the sidebar footer and opens the modal with the Image Studio card.
- Confirm the auto-accept image generation toggle in Settings shows the warning text.
- Confirm that the bug report queued message says "Will retry the next time the app talks to GitHub."
- Confirm that feedback actions show "Saved locally" for queued reports.
