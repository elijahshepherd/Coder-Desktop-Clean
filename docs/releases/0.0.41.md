# Bug Fixes

## Sub Apps modal now opens

The Sub Apps button in the sidebar footer dispatched a `sub-apps:open` custom event, and `App.tsx` correctly listened for it to open the modal. However, the `SubAppsModal` component itself also listened for the same event and called `onClose()` immediately after the parent opened it. This race condition caused the modal to close instantly. The rogue event listener has been removed.

## Image generation animation mockup text removed

The `black-and-white-spews.html` animation file contained a full "Mockup" UI with texture upload, clear, and clean-view controls, plus the text "Drag 7 red nodes to warp perspective freely. The center dot marks the cube corner. Press SPACE for clean view." This was leftover from a Three.js perspective-warp demo used during early prototyping. The entire mockup section has been replaced with a clean "Generating" heading and "Creating your image..." description. Unused CSS and JavaScript for the removed functionality has also been cleaned up.

## Edit button now works

The `onEditImage` callback was defined and the UI was built, but no parent component ever passed the prop to `ImageGenerationCard`. The `onEditImage` prop has now been wired through `MessageBubble` → `ChatWorkspace`, and a `handleEditImage` callback sends a follow-up message to the AI with the image prompt, triggering a new image generation.

## Feedback label no longer stuck on "Sending feedback"

When reports had a queued status without an `issueUrl`, the feedback actions component incorrectly showed "Sending feedback" indefinitely. Now correctly shows "Saved locally".

## Bug reports no longer require tokens

Previously, if no `CODER_DESKTOP_GITHUB_TOKEN`, `GH_TOKEN`, etc. environment variable was set, bug reports were queued locally and never sent to GitHub. Now the `gh` CLI (GitHub CLI) is used as a fallback to create issues automatically. The `gh` executable is checked in PATH and called via `execFile` with a temp file for the issue body. If `gh` also fails, the report is queued locally as before.

# Improvements

## Small icon sizes on image generation cards

The Pencil (Edit) and Download button icons were 16px. Increased to 18px for better visibility.

## Reduced compactness in Settings panel

Users reported that text, buttons, and all elements in the Settings panel were too compact. Spacing has been significantly increased across all settings-related components:

# New Features

## AS (Automatic Stoppers) maintenance system

The Automatic Stoppers system polls GitHub issues every 5 minutes for maintenance-format issues. When an open issue with a title matching `L:N T: Maintenance` is found, the system:

1. Parses the issue body for `ESID`, `OF`, `AL`, and `NOTE` fields
2. Checks if the platform matches (`ALL`, `Windows`, or `macOS`)
3. Writes a `maintenance/active.lock` file to the app data directory
4. Updates the `AppState.maintenance` field and pushes to the renderer
5. Blocks new `chat:send` messages when maintenance is active

When the maintenance issue is closed or no longer matches, the lock file is cleared and normal operation resumes.
