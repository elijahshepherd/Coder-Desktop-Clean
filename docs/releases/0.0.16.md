# Release Summary

## Overview

Coder Desktop 0.0.16 focuses on Windows update relaunch and exact logo source. This update matters because it improves the parts of the desktop app that users touch while building, testing, updating, or managing local AI-assisted work.

This version continues the local-first direction of Coder Desktop by tightening quality, release consistency, documentation, and day-to-day reliability. Users should expect a more predictable app surface and developers should have a clearer record of what changed in this version.

---

# Major Highlights

> This release includes three major highlights for users, testers, and developers.

## Windows Relaunch Repair

### This highlight covers the main Windows update relaunch and exact logo source work completed in this release.

* Added an icon generation script for rebuilding desktop package icons from the Coder Desktop SVG.

* Added updater tests for installer relaunch script behavior and fallback executable paths.

* Changed Windows installer relaunch behavior to try the current executable path and standard user install folders.

* Changed packaged icon generation so PNG, ICO, and ICNS assets come from the same SVG source.

* Fixed Windows installer updates so Coder Desktop relaunches after the silent installer finishes.

---

## Exact Logo Source

### This highlight explains the reliability, safety, and user-confidence improvements included in the release.

* Fixed Windows installer updates so Coder Desktop relaunches after the silent installer finishes.

* Fixed updater scripts so they no longer wait on the parent process, which could make the app close and appear to do nothing.

* Fixed the active Coder logo source so the renderer uses the exact provided SVG mark with dark and light mode inversion.

* Kept updater relaunch scripts local to the user's computer and avoided adding any remote execution path.

* Improved updater logging by writing installer handoff details to a local temporary update log.

# Upgrade Guidance

## Before Updating

* Close any running Coder Desktop windows before installing the new version.

* Keep a copy of important project work in source control or another backup before changing local tooling.

* Use the matching GitHub Release page or versioned download folder for the release artifact.

## After Updating

* Open Coder Desktop and confirm the app reports the expected version.

* Check provider settings, workspace access, and security controls if this release changed those areas.

* Run the workflow that mattered for the update, such as chat, local tools, settings, updates, or packaging.
