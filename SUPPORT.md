# Support

Support for Coder Desktop is focused on clear reports, release clarity, documentation, and reproducible product issues. The repository is read-only for public contribution by default, so support does not include general pull request review.

## Where To Ask

Use GitHub issues for:

- Bug reports.
- Release download problems.
- Packaging problems.
- Documentation problems.
- Provider setup confusion.
- Reproducible UI problems.
- Questions about expected app behavior.

Use the security reporting process in [SECURITY.md](SECURITY.md) for vulnerabilities or sensitive reports.

## Helpful Bug Reports

A helpful bug report includes:

- What you were trying to do.
- What happened.
- What you expected.
- Steps to reproduce.
- Operating system.
- Whether you used a packaged download or development mode.
- App version, release tag, or commit if known.
- Screenshots or logs when useful.

Please remove secrets before sharing logs or screenshots.

## Release Download Issues

For download issues, include:

- Release tag.
- Artifact name.
- Browser or download method.
- File size you received.
- Whether the app launched.
- Any operating system warning shown.
- Whether the file came from the official GitHub Release.

This helps separate packaging problems from local machine or browser download problems.

## Provider Issues

For provider setup issues, include:

- Provider name.
- Model name.
- Base URL.
- Whether the API key is present.
- The visible error message.

Do not include the raw API key.

## What Support Does Not Cover

Support does not cover:

- Reviewing uninvited pull requests.
- Debugging private projects without reproducible details.
- Guaranteeing SmartScreen reputation without a signing and reputation process.
- Supporting unofficial builds from unknown sources.
- Recovering lost API keys or provider account access.
