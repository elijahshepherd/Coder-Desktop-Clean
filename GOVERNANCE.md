# Governance

Coder Desktop is maintained by Elijah Shepherd. Project direction, release decisions, security posture, design standards, and contribution policy are controlled by the maintainer.

The repository is source-available for transparency and public inspection. It is read-only for public contribution unless a change is explicitly invited.

## Maintainer Authority

The maintainer decides:

- Product direction.
- Feature priorities.
- Security posture.
- Release timing.
- Release artifact policy.
- UI and documentation standards.
- Whether outside contributions are invited.
- Whether an issue remains open or closed.

This keeps the project focused and reduces accidental churn.

## Decision Principles

Project decisions should favor:

- Local-first behavior.
- Clear desktop app boundaries.
- Calm and professional UI.
- Explicit security controls.
- Real release downloads.
- Stable documentation.
- Direct ownership.

## Public Participation

Public participation is welcome through issues and reports. Public participation does not imply decision rights or acceptance of requested changes.

Helpful participation includes:

- Reproducible bug reports.
- Clear documentation feedback.
- Release download reports.
- Security reports through the proper channel.
- Thoughtful feature feedback.

## Release Governance

Official releases should be published through the official repository and GitHub Releases. Release folders should contain real artifacts and manifests. Placeholder release files should not be used as public downloads.

The maintainer controls release tags and release assets.

## Changes To Governance

This governance model may change as the project matures. Until it changes, Coder Desktop remains maintainer-led and read-only for public contribution by default.
