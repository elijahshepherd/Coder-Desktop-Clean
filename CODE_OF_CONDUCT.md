# Code Of Conduct

Coder Desktop is read-only for public contribution by default, but public issue discussions and project communication should still be respectful, clear, and useful.

## Expected Behavior

Participants are expected to:

- Be respectful.
- Stay focused on the topic.
- Share clear reproduction steps when reporting problems.
- Avoid personal attacks.
- Avoid harassment.
- Avoid posting secrets or private information.
- Respect maintainer decisions.
- Assume documentation can improve without treating people poorly.

## Unacceptable Behavior

Unacceptable behavior includes:

- Harassment or intimidation.
- Personal insults.
- Threats.
- Spam.
- Repeated off-topic comments.
- Posting private information.
- Posting secrets or credentials.
- Demanding maintainer action as if it is owed.
- Opening pull requests after being told they are not accepted by default.

## Issue Discussions

Good issue discussions should include useful details and stay connected to the report. If a discussion becomes unclear, the maintainer may ask for a focused reproduction, close the issue, or redirect the conversation.

## Enforcement

The maintainer may remove comments, close issues, limit participation, or block users when behavior harms the project or the people using it.

The goal is not to make discussion rigid. The goal is to keep the project readable, calm, and constructive.
