# Old Downloads

This folder is reserved for archived public release artifacts when older versions are moved out of the main [downloads](../downloads) folder.

The goal is to keep a public history of previous release files without confusing users about which artifacts are current.

## Archive Purpose

The archive can be used for:

- Older application downloads.
- Older manifests.
- Older release package documentation.
- Public comparison between releases.
- Recovery when a user needs a previous build.

## Archive Rules

Archived release folders should:

- Keep the original version number.
- Keep the original artifact names.
- Keep the original manifest when available.
- Avoid empty or temporary files.
- Avoid changing binary contents after archival.
- Link back to the matching release notes when possible.

## Current Downloads

Current public release artifacts belong in [downloads](../downloads). Older artifacts may move here when the project decides to keep the main download folder focused on recent releases.

## Git LFS

Archived binary artifacts may also use Git LFS. If a local checkout shows pointer files, use the official GitHub Release when a real download is needed.
